## Section 1 — The shift

Story points were built to estimate human effort.

But AI changed the bottleneck.

What matters now is not just how long work takes. It is:

- how much context an agent needs
- how many iterations are likely
- how much human review is required
- how risky rollout will be

CIRK models that reality.

---

## Section 2 — The model

### C — Context
How much understanding the agent needs

### I — Iteration
How many cycles are likely needed to converge

### R — Review
How much human validation is required

### K — Integration Risk
How risky or sensitive deployment is

---

## Section 3 — Example

**Task:** Build login API

```
C2 I2 R3 K1
```

Implication:

- the agent can prepare implementation
- human review is mandatory
- auto-merge is disabled

## Section 4 — Why it matters

AI writes code.
CIRK decides how that work should run.

It helps teams define:

- autonomy boundaries
- review rules
- deployment controls
- split requirements

---

## Section 5 — Open standard

CIRK is not a product.
It is an open standard for execution governance in AI-native software development.

It can be adopted in issue trackers, coding agents, pull request workflows, internal platforms, or governance layers such as Orbit618.
