# CIRK in 60 seconds

CIRK replaces effort-based estimation with execution clarity.

AI writes code. CIRK defines how that work should run.

---

## Step 1 — Pick a task

Example:

```
Build login API
```

---

## Step 2 — Score it

```
C2 I2 R3 K1
```

Meaning:

- **C2**: the agent needs moderate system context
- **I2**: some iteration is likely
- **R3**: human review is mandatory
- **K1**: rollout risk is low

---

## Step 3 — Apply execution policy

From that score, a team can decide:

- the agent may draft and implement
- a human must review before merge
- deployment can remain lightweight

---

## Step 4 — Use CIRK consistently

Once every task gets a CIRK score, teams gain a shared language for:

- autonomy
- review
- deployment safety
- execution control

---

## Rule of thumb

- **High R** → more human review
- **High K** → tighter deploy controls
- **High I** → more draft-first behavior
- **High C** → more context preparation

---

## What CIRK is not

CIRK is not:

- a time estimate
- a velocity metric
- a sprint planning abstraction

CIRK is an execution standard for AI-native development.
